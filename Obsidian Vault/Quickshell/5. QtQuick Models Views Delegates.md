# QtQuick Models, Views & Delegates

> [!tip] Step 5 · Level 1-2
> How to display lists of things. Used by system tray, workspace indicators, and app launchers.

---

## The Pattern

QML separates data (model), appearance (delegate), and layout (view):

```
Model  ──feeds──►  View  ──creates──►  Delegate (x N)
```

This is called **Model-View-Delegate** (MVD).

---

## Repeater

The simplest view. Creates N delegates in a row.

```qml
Column {
  Repeater {
    model: 5
    delegate: Text {
      text: "Item " + index
    }
  }
}
```

- `index` — built-in, 0-based position
- `modelData` — the data at this index (depends on model type)

---

## ListView

Scrollable list. Only creates delegates that are visible (efficient).

```qml
ListView {
  model: myModel
  delegate: Rectangle {
    width: parent.width; height: 40
    Text { text: modelData }
  }
  spacing: 4
}
```

Key properties:
- `orientation: ListView.Horizontal` — horizontal layout
- `snapMode: ListView.SnapToItem` — snap scrolling
- `highlight: Rectangle {}` — highlight item
- `currentIndex` — track selection

---

## GridView

Grid of items (like an icon view).

```qml
GridView {
  model: myModel
  delegate: myDelegate
  cellWidth: 100; cellHeight: 100
}
```

## Models

### Integer Model

```qml
model: 5  // creates 5 delegates with index 0-4
```

### JavaScript Array

```qml
model: ["Apple", "Banana", "Cherry"]
// delegate gets modelData = each string
```

### ListModel (dynamic)

```qml
ListModel {
  id: fruitModel
  ListElement { name: "Apple"; color: "red" }
  ListElement { name: "Banana"; color: "yellow" }
}
```

Then in delegate:
```qml
delegate: Text {
  text: name        // name comes from ListElement
  color: color      // color comes from ListElement
}
```

### QML Object Model

```qml
ObjectModel {
  id: objectModel
  Text { text: "Item 1"; color: "red" }
  Rectangle { width: 50; height: 50; color: "blue" }
}
```

Objects are the delegates directly — no extra delegate wrapper.

## How Your Config Uses MVD

### System Tray (TrayGroup.qml)

```qml
Repeater {
  model: SystemTray  // from Quickshell.Services.SystemTray
  delegate: TrayIcon {
    required property SystemTrayItem modelData
    // modelData.id, modelData.icon, modelData.tooltip, modelData.menu
  }
}
```

### Workspaces (WorkspacesWidget.qml)

Uses a `ListModel` populated by parsing `hyprctl` output, with `Repeater` for the workspace pills.

### QuickApps (shell.qml)

Reads JSON → JavaScript array → positions items on a semicircle using manual math (not a standard view).

---

## Key Rules

1. **Always use `required property`** in delegates — makes dependencies explicit
2. **Never put `id` in delegates** — IDs must be unique, delegates are created N times
3. **Use `modelData`** for simple arrays, **named properties** for ListModel
4. **Use `Component.onCompleted`** for non-binding setup in delegates
