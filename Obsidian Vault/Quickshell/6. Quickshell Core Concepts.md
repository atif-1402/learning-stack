# Quickshell Core Concepts

> [!tip] Step 6 · Level 2
> Quickshell is a QtQuick-based toolkit for building desktop shell components — bars, widgets, lockscreens, OSDs, launchers.

---

## How It Works

1. You write `.qml` files with Quickshell-specific types
2. Run with `qs -c myconfig` or `qs -p file.qml`
3. **Hot reload** — save a file, the UI updates live

---

## Config Discovery

```
~/.config/quickshell/
├── bar/
│   └── shell.qml        # entry point
├── clock/
│   └── shell.qml
└── ...
```

- Each subfolder with `shell.qml` = a config
- `qs -c bar` runs the `bar` config
- `qs -p path/to/file.qml` runs anything

## Window Types

### PanelWindow

Docked bar/panel. Reserves screen space (`exclusiveZone`). Anchors to edges.

```qml
PanelWindow {
  anchors { top: true; left: true; right: true }
  implicitHeight: 28
  color: Theme.background
  WlrLayershell.layer: WlrLayer.Top
  exclusiveZone: implicitHeight
}
```

Used by: [[17. Bar Component]], [[18. Clock Component]]

### FloatingWindow

Regular floating window, no edge anchoring.

```qml
FloatingWindow {
  width: 200; height: 100
  visible: true
}
```

### PopupWindow

Anchored popup (tooltips, menus, dropdowns).

```qml
PopupWindow {
  PopupAnchor {
    anchors {
      window: parent
      edge: Qt.BottomEdge
    }
  }
}
```

Used by: [[17. Bar Component]] tooltips and tray menus

### QsWindow

Base class for all Quickshell windows. Has `screen` property.

## Shell Structure Types

### ShellRoot

Top-level container. Like `Window` for regular Qt apps, but for Quickshell configs.

```qml
ShellRoot {
  Variants {
    model: Quickshell.screens
    Bar {}
  }
}
```

### Scope

A non-visual grouping container. Items inside are all default-property children (no visual layout). Used to share state across components.

```qml
Scope {
  id: root
  property string time

  Process { ... }   // shared process
  Timer { ... }     // shared timer

  Variants {
    model: Quickshell.screens
    PanelWindow { ... }
  }
}
```

### Singleton

A type with exactly one instance, accessible from anywhere.

```qml
pragma Singleton
import Quickshell
Singleton {
  property color background: "#1e1e2e"
  function fg(a) { ... }
}
```

Access: `Theme.background`, `Theme.fg(0.5)`

Used by: [[22. Theme System and Omarchy Integration]]

### Variants

Creates one instance of a delegate per item in a model. The key multi-monitor pattern.

```qml
Variants {
  model: Quickshell.screens
  PanelWindow {
    required property var modelData
    screen: modelData
    // ... one bar per screen
  }
}
```

## Process & IO

### Process

```qml
Process {
  command: ["date"]
  running: true
  stdout: StdioCollector {
    onStreamFinished: root.time = this.text
  }
}
```

### StdioCollector

Captures stdout. Fires `streamFinished` when done.

### Timer (QtQuick)

```qml
Timer {
  interval: 5000
  running: true
  repeat: true
  onTriggered: myProcess.running = true
}
```

## Layering (WlrLayershell)

| Layer | Z-order | Used By |
|-------|---------|---------|
| `WlrLayer.Background` | Very back | Floating clock background |
| `WlrLayer.Bottom` | Below windows | [[18. Clock Component]] |
| `WlrLayer.Top` | Above windows | [[17. Bar Component]] |
| `WlrLayer.Overlay` | Always on top | [[20. Frame Component]], [[19. OSD Component]] |
