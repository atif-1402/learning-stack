# Config Architecture Overview

> [!tip] Step 8 · Level 2
> Four independent Quickshell processes + a launcher = your desktop shell.

---

## Launch Flow

`start-qs.sh` kills old `quickshell` processes and starts four in parallel:

```bash
pkill -x quickshell 2>/dev/null
sleep 0.5
quickshell -c ~/.config/quickshell/bar &
quickshell -c ~/.config/quickshell/clock &
quickshell -c ~/.config/quickshell/frame &
quickshell -c ~/.config/quickshell/osd &
```

Each process loads its own `shell.qml` entry point. They don't share memory but they all read the same [[22. Theme System and Omarchy Integration]] file.

## Shared Theme File

Every component reads `~/.config/omarchy/current/theme/colors.toml` via shell commands.

See [[22. Theme System and Omarchy Integration]] for details.

## IPC: OSD Named Pipe

The OSD component uses a named pipe at `/tmp/qs-osd-pipe`:

```
osd/scripts/osd-volume    ──write──►  /tmp/qs-osd-pipe  ──read──►  OsdWindow.qml
osd/scripts/osd-brightness──write──►  /tmp/qs-osd-pipe  ──read──►  OsdWindow.qml
osd/scripts/osd-mic       ──write──►  /tmp/qs-osd-pipe  ──read──►  OsdWindow.qml
osd/scripts/osd-media     ──write──►  /tmp/qs-osd-pipe  ──read──►  OsdWindow.qml
```

See [[23. IPC and Named Pipes]].
