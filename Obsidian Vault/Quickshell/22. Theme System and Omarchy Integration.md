# Theme System & Omarchy Integration

> [!tip] Step 22 · Level 3
> Every Quickshell component reads colors from a shared theme file. This is the Omarchy theme system.

---

## Theme Source

All components read from:
```
~/.config/omarchy/current/theme/colors.toml
```

File format (TOML):
```toml
background = "#1e1e2e"
foreground = "#cdd6f4"
accent = "#fab387"
color8 = "#45475a"
```

The `omarchy` CLI manages themes. Switching themes updates this file, and Quickshell picks up the change within 2 seconds.

---

## How Each Component Reads It

| Component | Approach | Poll Interval | Colors Used |
|-----------|----------|---------------|-------------|
| **bar/Theme.qml** | `Process { command: ["cat", "..."] }` + regex | 2s | `background`, `foreground` + derived |
| **clock/shell.qml** | `grep` + `cut` in bash | On settings open | `foreground`, `accent` |
| **frame/shell.qml** | `grep` + `cut` in bash | 2s | `background` |
| **osd/OsdWindow.qml** | `Process` + TOML regex | 2s | `background`, `foreground`, `accent` |
| **quickapps/shell.qml** | `Process` + TOML regex | On launch | `background`, `foreground`, `accent`, `color8` |

## Theme.qml (Bar) — Deep Dive

`bar/Theme.qml` is a [[6. Quickshell Core Concepts#Singleton]]:

```qml
pragma Singleton
import Quickshell
Singleton {
  property color background: "#1e1e2e"
  property color foreground: "#cdd6f4"

  // Alpha helpers
  function fg(a) { return Qt.rgba(foreground.r, foreground.g, foreground.b, a) }
  function bg(a) { return Qt.rgba(background.r, background.g, background.b, a) }

  // Derived colors
  property color tooltipBackground: bg(0.8)
  property color groupHover: Qt.rgba(foreground.r, foreground.g, foreground.b, 0.05)
  property color pillActive: foreground
  property color pillInactive: fg(0.3)
  property color pillOccupied: accent
}
```

Access from any widget: `Theme.background`, `Theme.fg(0.5)`

## Live Theme Updates

Flow:
1. User runs `omarchy-theme-set <name>`
2. Omarchy updates `~/.config/omarchy/current/theme/colors.toml`
3. Quickshell `Timer` fires, re-reads the file
4. Property bindings propagate the new colors to all widgets
5. UI updates instantly

No restart required.
