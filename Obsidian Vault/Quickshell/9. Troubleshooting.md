# Troubleshooting

> [!tip] Step 9 · Level 2
> Common errors and how to fix them.

---

## Quick Help

```
qs log -c bar         # watch bar config logs
qs log -c clock       # watch clock config logs
pkill quickshell      # kill all QS processes
```

---

## 1. "ReferenceError: X is not defined"

**Cause:** Trying to access an `id` or property that doesn't exist in current scope.

**Fix:**
- Are you referencing an `id` from inside a `Component` delegate? You can't — delegates are separate scopes.
- Use `required property` and pass values explicitly, or use a [[6. Quickshell Core Concepts#Singleton]].

```qml
// WRONG
Component {
  Text { text: root.title }  // root might not exist here
}

// RIGHT
Component {
  required property string title
  Text { text: title }
}
```

---

## 2. Widget shows on only one monitor

**Cause:** Missing `Variants` wrapper.

**Fix:**
```qml
// WRONG — single instance
PanelWindow { ... }

// RIGHT — one per screen
ShellRoot {
  Variants {
    model: Quickshell.screens
    PanelWindow {
      required property var modelData
      screen: modelData
      ...
    }
  }
}
```

---

## 3. "Property X not found"

**Cause:** Mistyped property name, or trying to write to a read-only property.

**Fix:**
- Check the type's documentation for exact property names
- `readonly property` cannot be assigned; use `property` instead

---

## 4. Process doesn't run or no output

**Check:**
```qml
Process {
  command: ["ls", "-la"]    // NOT "ls -la" as a single string
  running: true
  stdout: StdioCollector {
    onStreamFinished: console.log("output:", this.text)
  }
}
```

**Common mistakes:**
- `command: ["ls -la"]` → WRONG, should be `["ls", "-la"]`
- Forgetting `running: true`
- Forgetting `StdioCollector` on `stdout`
- Process output has trailing newline — use `.trim()`

## 5. File not found / path issues

```qml
// WRONG
Process { command: ["cat", "theme.toml"] }

// RIGHT — use absolute path
Process { command: ["cat", "/home/user/.config/omarchy/theme.toml"] }
```

Or use environment variables via bash:
```qml
Process { command: ["bash", "-c", "cat $HOME/.config/omarchy/theme.toml"] }
```

## 6. Colors don't update after theme switch

**Check:**
- Is the `Timer` running?
- Is the Process running?
- Are you updating the right property?
- Is the property actually bound (reactive)?

```qml
// WRONG — not reactive, runs once
property color bg: "#000"

// RIGHT — reactive binding
property color bg: "#000"
// then something updates it:
onStreamFinished: root.bg = parseColor(this.text)
```

---

## 7. System tray icons not showing

**Requirement:** The config must use `UseQApplication` pragma:

```qml
//@ pragma UseQApplication
import Quickshell
import Quickshell.Services.SystemTray
```

Without this, the system tray won't work. Only one QS process can have this.

---

## 8. Window not visible / wrong layer

```qml
// WRONG — might be behind windows
PanelWindow { ... }

// RIGHT — specify layer
PanelWindow {
  WlrLayershell.layer: WlrLayer.Top   // above normal windows
  // or WlrLayer.Overlay               // always on top
  // or WlrLayer.Bottom               // below windows
}
```

---

## 9. "Module not found" / import errors

**Check:**
- Module name starts with uppercase: `import Quickshell`, not `import quickshell`
- Type name starts with uppercase: `PanelWindow`, not `panelWindow`
- File-based types need uppercase filename: `MyWidget.qml`, not `mywidget.qml`

---

## 10. Hot reload not working

- Quickshell watches `.qml` files in your config directory
- It does **not** reload scripts or config files — only QML
- If UI doesn't update, check `qs log -c <name>` for errors
- Sometimes a full restart is needed: `pkill quickshell && start-qs.sh`

## Debugging Tips

### Check your config:

```bash
qs log -c bar           # watch in real-time
pkill -x quickshell     # full restart
```

### Quick test a snippet:

```bash
qs -p /tmp/test.qml     # run any file standalone
```

### Console logging in QML:

```qml
Component.onCompleted: console.log("widget loaded")
onSomething: console.log("something happened", value)
```

Logs appear in `qs log -c <name>` output.

## Common QML Gotchas

| Gotcha | Explanation |
|--------|-------------|
| `color: "red"` not `color: "#ff0000"` | Both work, but named colors are limited |
| `anchors.centerIn: parent` silently ignored if parent has no size | Ensure parent has `width`/`height` |
| `Behavior on X` must be direct child of the animated object | Not inside a nested Item |
| `Timer` won't fire if `interval: 0` | Minimum 1ms |
| Properties are case-sensitive | `onclicked` ≠ `onClicked` |
