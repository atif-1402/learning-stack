# QML Language Basics

> [!tip] Step 3 · Level 1
> QML = Qt Modeling Language. A declarative language for defining UIs. Think JSON + JavaScript + reactive bindings.

---

## File Structure

```qml
import QtQuick           // imports
import Quickshell as QS  // namespaced import

// Root object (every file has one)
Item {
  id: root               // lowercase id, file-scoped
  property int count: 0  // property definition
  width: 200             // property binding
}
```

---

## Objects

```qml
TypeName {
  id: myId
  property: value
}
```

- Type names **must start with uppercase**
- Every file with an uppercase name becomes a reusable type (e.g. `MyWidget.qml` → `MyWidget {}`)
- Objects can contain properties, functions, and signals

---

## Properties

### Property Definitions

```qml
[required] [readonly] [default] property <type> <name>[: binding]

property int count: 0
readonly property string label: "hello"
required property var modelData    // must be set by user
```

### Property Bindings (Reactive)

```qml
Text {
  text: "clicks: " + root.count   // auto-updates when root.count changes
}
```

Bindings re-evaluate whenever any dependency changes. This is the **core reactivity model** of QML.

### The `id` Property

Every object can have an `id` (lowercase, unique per file). Used to refer to the object anywhere in the file.

```qml
Text { id: myText; text: "hi" }
Button { onClicked: myText.color = "red" }
```

### Property Access Scopes

You can access:
- Properties of the current object (`this.property` or just `property`)
- Properties of the **root object** of the current file (by name)
- Any object by its `id`

## Functions

```qml
function double(x: int): int {
  return x * 2
}
```

Functions are reactive — if a function reads `root.count` and `count` changes, anything bound to the function re-evaluates.

### Lambdas

```qml
property var op: (x) => x * 2
property var op2: x => x * 2
property var op3: (a, b) => { return a + b }
```

---

## Signals

### Definition

```qml
signal clicked(x: int, y: int)
```

### Handlers

```qml
Button {
  onClicked: console.log("clicked!")
}
```

Every signal `foo` gets an implicit `onFoo` handler property.

### Property Change Signals

```qml
onCountChanged: console.log("count is now", count)
```

### Connections (for singletons or indirect targets)

```qml
Connections {
  target: MySingleton
  function onDataChanged() { ... }
}
```

## Types (Creating Reusable Components)

Every `.qml` file with an uppercase name = a type.

```qml
// MyButton.qml
Rectangle {
  required property string label
  Text { text: parent.label }
}
```

```qml
// Usage
MyButton { label: "Click me" }
```

### Inline Components (v0.3.0)

```qml
Item {
  component HighlightedText: Text {
    color: "red"
    font.bold: true
  }
  HighlightedText { text: "Important" }
}
```

### Singletons

```qml
pragma Singleton
import Quickshell
Singleton {
  property string appName: "MyApp"
}
```

Access from anywhere: `AppName.appName`

---

## The Default Property

Types can declare a default property — children that don't specify a property name get assigned here.

```qml
Item {
  // children go into Item's default property (data/children)
  Text {}
  Rectangle {}
}
```

---

## Key Rules

| Rule | Example |
|------|---------|
| Type names = Uppercase | `Rectangle`, `PanelWindow` |
| Property names = lowercase | `width`, `text`, `myProp` |
| ids = lowercase | `id: root` |
| `required` enforces | `required property int count` |
| Semicolons optional | `width: 100` or `width: 100;` |
