# Frame Component

> [!tip] Step 20 · Level 3
> Draws a rounded-rectangle border around the active window. Uses `Canvas` 2D drawing with `destination-out` compositing to punch a hole through a colored rectangle.

---

## Files

| File | Purpose |
|------|---------|
| `frame/shell.qml` | Entry + drawing logic (141 lines) |
| `frame/config.json` | `radius` and `borderWidth` settings |

---

## How It Works

```qml
PanelWindow {
  WlrLayershell.layer: WlrLayer.Overlay
  // Covers entire screen
  // Draws a colored rect with a transparent hole punched out
  // The hole = active window bounds + border offset
}
```

Drawing technique:
1. Fill whole window with color (from theme)
2. Set `Canvas.context.compositingOperation = Canvas.destination-out`
3. Draw a rounded rectangle matching the active toplevel geometry
4. Result: colored border around the window, transparent everywhere else

---

## Config (config.json)

```json
{
  "radius": 20,
  "borderWidth": 1
}
```

- `radius` — corner radius of the border
- `borderWidth` — thickness of the border

Uses `FileView` + `JsonAdapter` to watch for file changes and update live.

---

## Active Window Tracking

```qml
ToplevelManager {
  onActiveToplevelChanged: {
    // Update frame position/size to match new active window
    canvas.requestPaint()
  }
}
```

- Reads `ToplevelManager.activeToplevel` for geometry
- Hides when active window is fullscreen (no border needed)

---

## Theme

Reads `background` color from `~/.config/omarchy/current/theme/colors.toml` every 2 seconds.

---

## Key Patterns

1. **Canvas 2D** — low-level drawing API for the border effect
2. **Destination-out compositing** — punches a transparent hole through a colored layer
3. **File watching** — `FileView` + `JsonAdapter` for live config.json reload
4. **Per-screen** — one frame window per monitor
