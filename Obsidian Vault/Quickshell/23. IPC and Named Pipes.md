# IPC & Named Pipes

> [!tip] Step 23 · Level 3
> How the OSD shell scripts communicate with the Quickshell OSD window.

---

## The Problem

Shell scripts (keyboard shortcuts) need to show visual feedback. Quickshell can read things, but how does a bash script tell Quickshell "show volume 45%"?

---

## Solution: Named Pipe

A FIFO (named pipe) at `/tmp/qs-osd-pipe`:

- Created by `OsdWindow.qml` on startup
- Shell scripts **write** to it
- Quickshell **reads** from it
- Each message is one line

---

## Message Protocol

```
<type>:<value>
```
or
```
<type>:<action>:<extra1>:<extra2>
```

### Message Types

| Type | Example | What happens |
|------|---------|--------------|
| `volume:45` | `volume:45` | Show speaker icon + 45% bar |
| `volume:muted` | `volume:muted` | Show muted icon + red bar |
| `brightness:80` | `brightness:80` | Show sun icon + 80% bar |
| `mic:muted` | `mic:muted` | Show "Muted" |
| `mic:unmuted` | `mic:unmuted` | Show "Active" |
| `media:play:Title:Artist` | `media:play:Bohemian Rhapsody:Queen` | Show now playing |

## Flow Diagram

```
WM Keybind
    │
    ▼
osd-volume up          osd-brightness down    osd-mic toggle        osd-media next
    │                       │                     │                     │
    ▼                       ▼                     ▼                     ▼
wpctl set-volume 5%+   brightnessctl set 5%-   wpctl set-mute toggle   playerctl next
    │                       │                     │                     │
    └───────────────────────┴─────────────────────┴─────────────────────┘
                                    │
                                    ▼
                      echo "volume:45" > /tmp/qs-osd-pipe
                                    │
                                    ▼
                         OsdWindow.qml reads
                         Parses "volume:45"
                         Animates OSD pill
                         Auto-hides after 2s
```

## Implementation in QML

```qml
// OsdWindow.qml
Socket {
  // Connects to /tmp/qs-osd-pipe
  // Reads lines
  // Splits by ":"
  // Sets OSD type, value, text
  // Triggers show animation + timer
}
```

---

## Why Not D-Bus?

Named pipes are:
- Simpler (no D-Bus dependencies)
- Faster (no message bus overhead)
- Shell-script friendly (just `echo > pipe`)
- Works without a D-Bus session

---

## Key Pattern

**One-way, fire-and-forget IPC.** The scripts don't wait for a response. The OSD window shows the state for 2 seconds and hides. No acknowledgement needed.
