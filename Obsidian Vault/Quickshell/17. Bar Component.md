# Bar Component

> [!tip] Step 17 · Level 3
> The main top panel. 28px tall, anchored to top, sits on `WlrLayer.Top`. Shows status on the left, clock/workspaces in the center, system tray on the right.

---

## Files

| File | Purpose |
|------|---------|
| `bar/shell.qml` | Entry: `ShellRoot` + `Variants` per screen |
| `bar/Bar.qml` | `PanelWindow` definition |
| `bar/Theme.qml` | Singleton: reads `colors.toml` |
| `bar/modules/*.qml` | Individual widgets (17 files) |

---

## Bar.qml — Key Structure

```qml
PanelWindow {
  required property var modelData
  screen: modelData
  anchors { top: true; left: true; right: true }
  implicitHeight: 28
  WlrLayershell.layer: WlrLayer.Top
  exclusiveZone: implicitHeight
  color: Theme.background

  // LEFT: StatusGroup
  StatusGroup {}

  // CENTER: RowLayout
  RowLayout {
    anchors.centerIn: parent
    ClockWidget {}
    WorkspacesWidget {}
    CtlsGroup {}
  }

  // RIGHT: TrayGroup
  TrayGroup {}
}
```

**Key details:**
- `required property var modelData` — injected by `Variants` from `Quickshell.screens`
- `exclusiveZone: implicitHeight` — reserves 28px so maximized windows don't overlap
- `WlrLayer.Top` — sits above normal windows but below overlay

## Theme Integration

`bar/Theme.qml` is a [[6. Quickshell Core Concepts#Singleton]]. It:
- Runs `cat ~/.config/omarchy/current/theme/colors.toml` every 2 seconds
- Parses `background` and `foreground` with regex
- Provides helper functions `fg(a)` and `bg(a)` for alpha-adjusted colors
- Defines derived colors for workspace pills, tooltips, group boxes

See [[22. Theme System and Omarchy Integration]].

## Left Section — StatusGroup (`bar/modules/StatusGroup.qml`)

A [[GroupBox]] containing:

### BatteryWidget
- Reads `/sys/class/power_supply/BAT*/capacity` and `status` every 5s
- Pill-shaped progress bar
- Red ≤10%, yellow ≤20%, green when charging
- Tap animation with bounce + ripple effects

### Dot
- Just a `""` Nerd Font separator character

### ScreenRecordingWidget
- Runs `$OMARCHY_PATH/default/waybar/indicators/screen-recording.sh` every 2s
- Shows red icon when recording active
- Tap → `omarchy-capture-screenrecording`

### IdleWidget
- Runs `$OMARCHY_PATH/default/waybar/indicators/idle.sh` every 3s
- Shows icon when idle/DPMS is active
- Tap → `omarchy-toggle-idle`

### NotificationSilenceWidget
- Runs `$OMARCHY_PATH/default/waybar/indicators/notification-silencing.sh` every 3s
- Shows icon when DND is active
- Tap → `omarchy-toggle-notification-silencing`

### VoxtypeWidget
- Polls `voxtype status --extended --format json` every 1s
- Shows recording (red) or transcribing icon
- Left-click → `omarchy-voxtype-model`, right-click → `omarchy-voxtype-config`

### WeatherWidget
- Runs `$OMARCHY_PATH/default/waybar/weather.sh` every 60s
- Expects JSON `{"text":"..."}` output
- Tap → `notify-send` with details

### UpdateWidget
- Runs `omarchy-update-available` every 6 hours
- Shows refresh icon when updates available
- Tap → `omarchy-update` in floating terminal

## Center Section

### ClockWidget (`bar/modules/ClockWidget.qml`)
- Shows time (HH:MM) or date (dd Month Day yyyy)
- Toggle time/date on left-click
- Right-click → `omarchy-tz-select`
- Timer every 1s

### WorkspacesWidget (`bar/modules/WorkspacesWidget.qml`)
- Hyprland workspace switcher
- Pills for workspaces 1-5 + dynamic extras
- Uses `hyprctl clients -j` + Hyprland raw events for occupancy tracking
- Events: `openwindow`, `closewindow`, `movewindowv2`, `createworkspacev2`, `destroyworkspacev2`, `workspacev2`, `urgent`
- Click → `Hyprland.dispatch("workspace N")`

### CtlsGroup — Controls Group ([[GroupBox]])
- **AudioWidget** — polls `pactl` every 2s for volume + mute + port type
- **BluetoothWidget** — polls `bluetoothctl` every 5s, shows connected devices in tooltip
- **NetworkWidget** — polls `ip` + `iw` every 3s, detects WiFi vs Ethernet, signal bars

## Right Section — TrayGroup (`bar/modules/TrayGroup.qml`)

Custom system tray implementation:
- Binds to `SystemTray` model (from Quickshell.Services.SystemTray)
- Renders each `SystemTrayItem` as a clickable icon
- Right-click opens a `PopupWindow` menu with `QsMenuEntry` items
- Uses `HyprlandFocusGrab` for focus management
- Submenu support via `QsMenuAnchor`

## Utility Widgets

### BarIcon (`bar/modules/BarIcon.qml`)
Reusable clickable icon:
- Nerd Font character display
- Left-click + right-click command support via [[CommandRunner]]
- Hover opacity effect
- `BarTooltip` on hover

### BarTooltip (`bar/modules/BarTooltip.qml`)
Animated `PopupWindow`:
- Anchors to parent window bottom edge
- Bounce-in (`Easing.OutBack`) and fade-out (`Easing.InCubic`) animations
- Rounded rect with border

### GroupBox (`bar/modules/GroupBox.qml`)
Horizontal container with:
- `leftMargin: 4`, `rightMargin: 5` padding
- Hover highlight via `Theme.groupHover`

### CommandRunner (`bar/modules/CommandRunner.qml`)
Queue-based command executor:
- Runs `setsid -f bash -lc <cmd>` (detaches from QS, runs in new session)
- Captures stdout/stderr
- Sends notification on non-zero exit
- Drains queue sequentially

## Key Patterns

1. **Polling loop**: `Timer` → `Process.running = true` → `StdioCollector.onStreamFinished` → update property
2. **Theme reactivity**: [[22. Theme System and Omarchy Integration]] polls every 2s, colors update via property bindings
3. **Multi-monitor**: `Variants` + `screen: modelData`
4. **Icon font**: All icons are [[Nerd Font]] characters via `JetBrainsMono Nerd Font Propo`
