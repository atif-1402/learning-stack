# OSD Component

> [!tip] Step 19 · Level 3
> On-Screen Display overlay. Shows volume, brightness, microphone, and media playback status as an animated pill at the bottom of the screen.

---

## Files

| File | Purpose |
|------|---------|
| `osd/shell.qml` | Entry: `ShellRoot` + `Variants` per screen |
| `osd/OsdWindow.qml` | The OSD pill animation + IPC reader (291 lines) |
| `osd/scripts/osd-volume` | Volume up/down/mute via `wpctl` |
| `osd/scripts/osd-brightness` | Brightness up/down via `brightnessctl` |
| `osd/scripts/osd-mic` | Mic mute toggle via `wpctl` |
| `osd/scripts/osd-media` | Media play/pause/next/prev via `playerctl` |

---

## Architecture: IPC via Named Pipe

```
Keybind → osd-* script → writes to /tmp/qs-osd-pipe → OsdWindow reads → animates
```

The shell scripts are bound to keyboard shortcuts in your WM. They modify the system and send a message to the OSD pipe:

```
volume:45          → show volume bar at 45%
volume:muted       → show muted icon
brightness:80      → show brightness bar at 80%
mic:muted          → show "Muted"
mic:unmuted        → show "Active"
media:play:Title:Artist  → show Now Playing
```

## OsdWindow.qml — Key Structure

```qml
PanelWindow {
  WlrLayershell.layer: WlrLayer.Overlay
  anchors { bottom: true; horizontalCenter: parent.horizontalCenter }
  // Reads from /tmp/qs-osd-pipe via FileView or polling
  // Shows pill-shaped overlay with icon + progress bar + text
  // Auto-hides after 2 seconds
}
```

### Message Format

```
<type>:<value>
<type>:<detail>:<extra1>:<extra2>
```

| Type | Value | Display |
|------|-------|---------|
| `volume` | `0`-`100` or `muted` | Speaker icon + % bar; red if muted |
| `brightness` | `0`-`100` | Sun icon + % bar |
| `mic` | `muted` / `unmuted` | Mic icon + "Muted"/"Active" |
| `media` | `pause:Title:Artist` | Pause icon + title + artist |

## Animations

- Slides up from bottom
- Opacity fade in/out
- Width/height animated
- Auto-hides after 2 seconds


## Scripts Detail

### osd-volume
```bash
wpctl set-volume @DEFAULT_AUDIO_SINK@ 5%+    # up
wpctl set-volume @DEFAULT_AUDIO_SINK@ 5%-    # down
wpctl set-mute @DEFAULT_AUDIO_SINK@ toggle   # mute
# Writes "volume:<pct>" or "volume:muted" to pipe
```

### osd-brightness
```bash
brightnessctl set 5%+     # up
brightnessctl set 5%-     # down
# Writes "brightness:<pct>" to pipe
```

### osd-mic
```bash
wpctl set-mute @DEFAULT_AUDIO_SOURCE@ toggle
# Writes "mic:muted" or "mic:unmuted" to pipe
```

### osd-media
```bash
playerctl play-pause
playerctl next
playerctl previous
# Writes "media:<action>:<title>:<artist>" to pipe
```

## Theme

Reads `background`, `foreground`, `accent` from `~/.config/omarchy/current/theme/colors.toml` with the same TOML regex approach as `bar/Theme.qml`. Polls every 2 seconds.

---

## Key Patterns

1. **Named pipe IPC** — lightweight, no D-Bus needed
2. **Animated show/hide** — auto-dismiss after 2s
3. **Generic OSD** — same window handles 4+ different data types via message format
4. **Overlay layer** — always visible above windows
